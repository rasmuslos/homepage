Get support for AmpFin & ShelfPlayer

Thank you for using my iOS apps! I value your feedback and am here to assist you in any way I can. If you encounter any issues, have suggestions for improvement, or simply want to get in touch with me, there are two main support options available to you: sending me an email or opening a GitHub issue.

1. Sending an Email:

If you prefer to communicate via email, you can reach out to me directly at git@rfk.io. I am dedicated to addressing your concerns promptly and providing assistance as needed. When contacting me via email, please provide the following information:

  • App Version: Please include the version number of the app you are using.
  • Source of the App: Let me know where you downloaded or obtained the app (e.g., App Store, TestFlight).
  • Detailed Description: Provide a detailed description of the problem or feedback you wish to share. Include any steps to reproduce the issue, error messages encountered, and any other relevant information.
  • Screenshots: If possible, attach screenshots that illustrate the issue you are experiencing. This can greatly help in understanding and resolving the problem.

2. Opening a GitHub Issue:

Alternatively, you can open an issue on my GitHub repository to report bugs, suggest features, or discuss any aspect of my apps with me. To open a GitHub issue, follow these steps:

  • Visit the GitHub repository for AmpFin or ShelfPlayer
  • Click on the "Issues" tab near the top of the repository page.
  • Then, click the green "New issue" button.
  • Provide a descriptive title for your issue, along with a detailed description of the problem or feedback you'd like to share.
  • Include the following details:
  • App Version: Please include the version number of the app you are using.
  • Source of the App: Let me know where you downloaded or obtained the app (e.g., App Store, TestFlight).
  • Detailed Description: Provide a detailed description of the problem or feedback you wish to share. Include any steps to reproduce the issue, error messages encountered, and any other relevant information.
  • Screenshots: If possible, attach screenshots that illustrate the issue you are experiencing. This can greatly help in understanding and resolving the problem.

I actively monitor GitHub issues and strive to address them in a timely manner. Opening an issue on GitHub allows you to track the status of your request and engage in discussions with me.

Additional Information:

Before contacting me or opening a GitHub issue, please ensure that you are using the latest version of my app. Updates often include bug fixes and improvements that may resolve the issue you are experiencing.

Thank you for using my iOS apps and for helping me improve them! I appreciate your support and look forward to hearing from you.
